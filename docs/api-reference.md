# API Reference

Base URL: `http://localhost:8000`  
Interactive docs (Swagger UI): `http://localhost:8000/docs`

---

## Search

### `GET /search`

Hybrid semantic + keyword search over a collection.

**Query parameters:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `q` | string | ✅ | Search query |
| `collection` | string | ✅ | Collection name (matches `weaviate.collection` in config) |
| `limit` | int | — | Number of results (default: `api.default_limit`, max: `api.max_limit`) |
| `fields` | string | — | Comma-separated fields to return (default: `api.output_fields`) |
| `filter` | string | — | Structured filter: `Field:Value` or `Field1:Value1,Field2:Value2` |
| `min_score` | float | — | Minimum score threshold `0.0–1.0`; excludes lower results |

**Example:**

```bash
curl "http://localhost:8000/search?q=senior+engineer&collection=Employees&filter=Department:Engineering&min_score=0.6"
```

**Response:**

```json
{
  "results": [
    {
      "name": "Jane Smith",
      "role": "Senior Software Engineer",
      "department": "Engineering",
      "_score": 0.87
    }
  ],
  "took_ms": 42
}
```

**Notes:**
- Uses Weaviate hybrid search: BM25 (keyword) + `near_vector` (semantic) with `alpha=0.5`
- `_score` is higher = better (not distance)
- `filter` only works on `metadata_fields`; unknown fields return `422`
- Filter values may contain `:` — only the first colon is used as separator

## Sync

### `POST /sync`

Trigger an incremental sync (only new or changed records).

```bash
curl -X POST "http://localhost:8000/sync?collection=Products"
```

### `POST /sync/full`

Trigger a full re-index (drops collection, re-embeds all records).

```bash
curl -X POST "http://localhost:8000/sync/full?collection=Products"
```

### `GET /sync/status`

Get the status of the last sync run.

```bash
curl "http://localhost:8000/sync/status?collection=Products"
```

**Response:**

```json
{
  "status": "success",
  "records_synced": 1247,
  "took_ms": 38200,
  "started_at": "2026-05-19T10:30:00Z"
}
```

**Notes:**
- Sync runs in the background (non-blocking). The endpoint returns immediately.
- Only one sync can run at a time (concurrent requests return `409`).

## Collections

### `GET /collections`

List all available collections (based on sub-directories under `configuration/`).

```bash
curl http://localhost:8000/collections
```

**Response:**

```json
["Products", "Articles", "Employees"]
```

---

### `POST /collections/{name}/cache`

Purge all cached search results for one entity. **Admin only** (`require_admin` → 403 otherwise).

Use it after editing an entity's `config.yaml` / `synonyms.yaml` or its data: the exact-match
search cache (TTL 300s) is not auto-invalidated on file edits, so without a purge a stale
result can be served for up to the TTL. Fast synchronous operation (no background task).

```bash
curl -X POST http://localhost:8000/collections/Employees/cache \
  -H "Authorization: Bearer $TOKEN"
```

**Response:** `{"status": "purged", "collection": "Employees"}` — or `{"status": "noop", "detail": "cache disabled"}` when no cache is configured. Invalid name → `422`.

> In the UI: **Settings → Entities**, select an entity, **"Svuota cache"** in its info panel.

## Observability

### `GET /health`

Liveness check — verifies Weaviate connectivity.

```bash
curl http://localhost:8000/health
# 200 OK  →  {"status": "ok"}
# 503     →  {"status": "weaviate_unreachable"}
```

### `GET /info`

Returns runtime info about the active collection.

```bash
curl "http://localhost:8000/info?collection=Products"
```

**Response:**

```json
{
  "collection": "Products",
  "total_objects": 5432,
  "embedding_model": "qwen3-embedding:4b",
  "embedding_type": "ollama"
}
```

## Setup

### `POST /setup/suggest-config`

Upload a CSV and get an LLM-generated `config.yaml` suggestion (`text_fields`, `metadata_fields`, `id_field`, collection name).

```bash
curl -X POST http://localhost:8000/setup/suggest-config \
  -F "file=@./data/products.csv"
```

**Response:**

```json
{
  "suggested_config": {
    "weaviate": {
      "collection": "Products",
      "text_fields": ["description", "name"],
      "metadata_fields": ["price", "category", "sku"]
    },
    "source": {
      "id_field": "sku"
    }
  }
}
```

> This endpoint never auto-applies the suggestion. Review and edit `config.yaml` manually.

## Knowledge Graph

### `GET /graph/{collection}`

Compute a UMAP projection + HDBSCAN clustering of the collection's embeddings. Returns nodes, edges, and cluster labels (generated by LLM).

```bash
curl http://localhost:8000/graph/Products
```

**Response:**

```json
{
  "nodes": [
    {
      "id": "uuid-...",
      "x": 0.42,
      "y": -0.87,
      "cluster": 2,
      "props": { "name": "Widget A", "category": "tools" }
    }
  ],
  "edges": [
    { "source": "uuid-a", "target": "uuid-b" }
  ],
  "clusters": [
    { "id": 2, "name": "Hand Tools", "size": 143 }
  ]
}
```

**Notes:**
- Requires at least ~15 records for HDBSCAN to produce meaningful clusters
- Cluster names are generated by an Ollama LLM (`qwen2.5:3b` by default)
- `edges` come from UMAP's internal k-NN graph — no additional Weaviate queries

## Logs

### `GET /logs`

Retrieve sync history for a collection.

```bash
curl "http://localhost:8000/logs?collection=Products&limit=20"
```

**Response:**

```json
{
  "logs": [
    {
      "id": 1,
      "collection": "Products",
      "type": "full",
      "status": "success",
      "records_synced": 1247,
      "took_ms": 38200,
      "triggered_by": "manual",
      "started_at": "2026-05-19T10:30:00Z"
    }
  ]
}
```

## File Upload

### `POST /upload`

Upload a CSV or JSON file and trigger a sync for the specified collection.

```bash
curl -X POST http://localhost:8000/upload \
  -F "file=@./data/products.csv" \
  -F "collection=Products"
```
